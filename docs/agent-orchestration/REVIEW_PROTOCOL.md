# Review Protocol

1. Validate the task package and run local checks.
2. Launch fresh read-only Scientific and Artifact reviews with immutable inputs.
3. Validate and deduplicate `FINDINGS`; preserve the originating roles.
4. Resume the original Owner with actionable findings for any repairs.
5. Launch a fresh read-only Final Verifier to rerun acceptance evidence.
6. The Leader records PASS, PARTIAL, or FAIL and requests the required human checkpoint.

Reviewers report evidence and severity; they do not write fixes or accept results.
