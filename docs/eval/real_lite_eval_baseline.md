# Real-Lite Eval Baseline

## Goal

Phase 5d adds an offline, deterministic eval baseline for the real-lite
workflow output under:

```text
<OUTPUT_ROOT>
```

The baseline measures workflow completeness, artifact health, quality gate output, figure integrity, citation/reference structure, prompt leakage, evidence coverage, and safety boundaries.

## Why Custom First

This phase does not install or run promptfoo. A custom standard-library runner is easier to keep offline, deterministic, and safe while the workflow artifacts are still evolving. A promptfoo-compatible draft is included only as future reference.

## Metrics

| metric | purpose |
| --- | --- |
| `workflow_completeness` | checkpoint log exists, has 9 checkpoints, and each checkpoint records status, inputs, outputs, and human-review requirement |
| `artifact_completeness` | required stage artifacts exist and are non-empty |
| `quality_gate_health` | quality report is present, `pass` or `warn`, and has zero blocking errors |
| `figure_integrity` | figure manifest is non-empty and each figure has id, source pointer, and caption |
| `citation_and_reference_integrity` | final draft has citation callouts, ordered groups, and a References section |
| `prompt_leakage_absence` | final draft does not contain obvious process/prompt leakage terms |
| `evidence_coverage` | at least 3 selected papers are visible across stage outputs |
| `safety_boundary` | run summary confirms no network, PDF read, Qwen, MinerU API, upload, knowledge base, or image API usage |

## Pass/Fail Standard

The current expected fixture requires:

- `minimum_score`: 90
- all 8 required metrics must be `pass`

The real-lite v1 run currently scores 100 when generated by `make real-lite-e2e-check`.

## Run

```bash
python scripts/eval/run_eval_baseline.py \
  --output-root <OUTPUT_ROOT> \
  --baseline evals/baselines/real_lite_v1.yaml \
  --expected evals/fixtures/real_lite_expected_metrics.json \
  --output-json /tmp/real_lite_eval_report.json \
  --output-md /tmp/real_lite_eval_report.md \
  --strict
```

Or:

```bash
make eval-baseline-check
```

## Reports

Reports are generated under `/tmp`:

```text
/tmp/real_lite_eval_report.json
/tmp/real_lite_eval_report.md
```

Do not commit generated reports unless a release artifact is explicitly requested.

## Safety Boundary

- No full `chem_papers` scan.
- No PDF body read.
- No MinerU API.
- No Qwen call.
- No upload.
- No Bailian knowledge base.
- No image generation.
- No promptfoo install or run.

## Current Limits

- The baseline evaluates workflow artifacts, not final scientific quality.
- It uses static and heuristic checks only.
- It does not judge whether the chemistry synthesis is complete or novel.
- Promptfoo support is a draft only.
