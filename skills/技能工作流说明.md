# Skills 工作流说明

当前 skill 分两类。

## 写作前准备

这部分不放进主工作流，由用户按需单独执行：

```text
mineru-precise-parse-review-writer
review-metadata-prep
```

用途：

```text
PDF -> MinerU Markdown/content_list/images
MinerU 输出 -> review-library metadata
人工审核论文标题、摘要、关键词、标签和路径
```

审核页面：

```text
http://127.0.0.1:8765/library
```

## 主工作流入口

主工作流统一从这个 skill 进入：

```text
review-writing-orchestrator
```

它负责：

```text
检查项目当前状态
决定下一步调用哪个 skill
列出缺失产物
在人审节点暂停
```

状态脚本：

```bash
python <review_root>/skills/review-writing-orchestrator/scripts/project_status.py \
  --review-root <review_root> \
  --project-id <project_id>
```

## 主工作流顺序

```text
1. review-topic-paper-discovery
2. 人工 check discovery
3. review-literature-matrix-outline
4. 人工 check 文献矩阵和大纲
5. review-section-blueprint
6. 人工 check 章节论证蓝图
7. review-section-drafting-figure-picking
8. 人工 check 章节草稿和图表候选；正常流程应至少有核心章节图/反应 scheme 候选
9. review-figure-style-redraw
10. 人工 check 重绘图片；无成功重绘图片时不应直接合并成无图综述
11. review-draft-merge-polish
12. 人工 check 初稿
13. review-final-audit-release
14. 人工最终确认终稿
15. review-export-docx
16. 人工下载 final_draft.docx 并在 Word 中确认排版
```

## 阶段产物

```text
00_discovery/         主题、关键词、候选文献和人工筛选结果
01_matrix_outline/    深读笔记、文献矩阵、2-3 套大纲、章节论证蓝图
02_section_drafting/  章节任务、章节草稿、原文图候选
03_figure_redraw/     统一风格重绘图片和 manifest
04_first_draft/       合并后的初稿、merge report、remaining issues
05_final_audit/       内容检查、格式检查、终稿和发布报告
```

完整流程默认目标是带图综述。图片候选应在章节写作阶段完成，图片统一风格重绘在 `03_figure_redraw/` 完成，插入正文在 `04_first_draft/` 完成。

`review-section-blueprint` 中的段落规则采用 rule pack 结构。当前默认包是 `references/rule_packs/allenation/`，主要服务联烯化、联烯合成和炔丙醇/炔丙基衍生物综述。未来不同领域可以新增并注册自己的 rule pack。选定 rule pack 不仅用于生成章节论证蓝图，也必须在 `review-section-drafting-figure-picking` 正式写段落时读取。章节草稿应从 `section_blueprint.json` 的 claim、paper role、logic relationship 和 wording constraints 出发，按规则库完成首写和段落级自检。

页面对应顺序：

```text
Library -> Discovery -> Matrix -> Blueprint -> Sections -> Figures -> Draft -> Final
```

`Figures` 页面重点检查源图是否定位成功、是否完成统一风格重绘。若 `project_status.py` 报告 `figure_redraw_skipped` 或 `no_redrawn_figures`，不要直接进入无图终稿，除非用户明确要求跳过图片。

## 终点

主工作流以 `05_final_audit/final_draft.md` 和 `05_final_audit/release_report.md` 作为终点。
